# Docs

Este directorio centraliza la documentación del proyecto por áreas de trabajo.

## Estructura

- `business/`: estrategia comercial, propuesta de valor, pricing, ventas y posicionamiento
- `product/`: visión de producto, discovery, roadmap, requerimientos y UX
- `engineering/`: arquitectura, estándares técnicos, decisiones de ingeniería y operaciones
- `data/`: modelos de datos, analytics, eventos, reporting e infraestructura de datos
- `ai/`: estrategia de inteligencia artificial, prompts, evaluación e integración de modelos
- `operations/`: procesos internos, colaboración, delivery y documentación operativa

## Convención de nombres

Usar nombres descriptivos en `kebab-case`, por ejemplo:

- `saas-technical-partner-positioning.md`
- `mvp-scope-and-assumptions.md`
- `architecture-overview.md`

## Recomendación

Si un documento mezcla varias áreas, ubicarlo según su intención principal. Si nace para ventas, estrategia o conversión, normalmente pertenece a `business/`.
